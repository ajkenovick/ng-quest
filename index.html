Breve descripción de cómo instaló el servidor Web, ¿qué herramientas o comandos usó?, ¿qué inconvenientes encontró?
Para instalarlo sudo dnf install nginx
Para iniciar el servicio nginx sudo systemctl start nginx
Para iniciar automáticamente el servicio nginx con el sistema operativo sudo systemctl enable nginx
Para cambiar la configuración de puerto y de ruta de proyectos con el editor nano sudo nano /etc/nginx/nginx.conf
Para reiniciar el servicio nginx sudo systemctl restart nginx
Para recargar el servicio nginx sudo systemctl reload nginx
Para crear la carpeta nueva de proyecto sudo mkdir -p /home/ninja/projects/nginx
Clonar contenido de la carpeta por defecto de ngnix por la nueva carpeta de proyecto sudo rsync -av /usr/share/nginx/html "/home/ninja/projects/nginx"
Inconvenientes 1 Al momento de probar la ruta nueva de proyecto, me daba error 403 que decía que el archivo o la carpeta estaba perdida. Por eso se investigó y se hizo los cambios de permisos
Para cambiar los permisos de la carpeta de uusario ninja sudo chmod 711 /home/ninja
Para cambiar los permisos de la carpeta nueva de proyecto sudo chmod 755 /home/ninja/projects/nginx
Para cambiar los permisos de los archivos dentro de la carpeta nueva de proyecto sudo chmod 644 /home/ninja/projects/nginx/*
Inconvenientes 2 Al momento de probar el index.html que venía en la carpeta por defecto, estaba con link símbolico, por lo tanto se debía hacer el enlace al recién copiado index.html
Para dar enlace símbolico al archivo index.html al que viene por defecto ln -sfn /usr/share/testpage/index.html /home/ninja/projects/nginx/index.html
